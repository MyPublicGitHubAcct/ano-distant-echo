# CabinetIR.h — Speaker Cabinet IR Convolution

## Purpose

`CabinetIR` simulates the frequency response of a guitar speaker cabinet by convolving the input signal with a pre-designed 256-tap minimum-phase FIR impulse response. It is a standalone `Effect` subclass applied after the Overdrive output independently of the drive or waveshaper settings.

Three cabinet presets are available via the `CabinetType` enum:

| Preset | Label | Resonance | HF Rolloff | Character |
| --- | --- | --- | --- | --- |
| `OpenBack1x12` | 1×12 Open-Back | +6 dB at 120 Hz (Q=2) | −6 dB/oct above 4 kHz | Warm, open; classic British combo |
| `ClosedBack4x12` | 4×12 Closed-Back | +6 dB at 80 Hz (Q=2) | −3 dB/oct above 3 kHz | Tight low end, more present mids; stack cabinet |
| `Combo1x12` | 1×12 Combo | +6 dB at 180 Hz (Q=2) | −6 dB/oct above 5 kHz | Brighter, extended HF; American combo |

All three IRs are normalized to unity gain at 1 kHz (flat passband reference).

## IR Design

Each IR is a minimum-phase FIR generated by `python/cabinet_ir.py`. The design process:

1. Compute the target magnitude spectrum on a 512-point grid:
   - Lorentzian resonance peak at the preset's resonant frequency (Q=2, +6 dB).
   - First- or half-order rolloff above the preset's HF corner frequency.
2. Reconstruct the minimum-phase filter from the target magnitude via the real cepstrum:
   - Take `log(mag_full)`.
   - Compute the real cepstrum via `ifft(log_mag)`.
   - Apply the minimum-phase lifter: double positive-time cepstrum, zero negative-time, leave DC and Nyquist unchanged.
   - Exponentiate and take the inverse FFT; take the real part as the IR.
3. Truncate to 256 taps (minimum-phase energy is front-loaded; the tail is near zero).
4. Normalize so that `|H(1 kHz)| = 1.0`.

The resulting IRs are stored as `constexpr float` arrays in separate generated headers. Regenerate with:

```sh
uv run --project python python/cabinet_ir.py --generate-header --type 1x12   # → CabinetIR_data.h
uv run --project python python/cabinet_ir.py --generate-header --type 4x12   # → CabinetIR_data_4x12.h
uv run --project python python/cabinet_ir.py --generate-header --type combo  # → CabinetIR_data_combo.h
```

### Validation

`python/cabinet_ir.py --validate` confirms that the Python direct-form convolution matches `scipy.signal.fftconvolve` with a maximum sample error of **< 9.4e-17** (effectively machine precision). The C++ direct-form convolution is validated against the Python fftconvolve golden WAVs with a tolerance of **5e-4** via `make compare`.

## Sample-Rate Resampling

All three IRs are designed at 48 kHz. `CabinetIR::prepare()` calls `buildActiveIR()` to resample the selected base IR to the current host sample rate using linear interpolation. This keeps the resonance and rolloff frequencies correct at 44.1 kHz, 48 kHz, and 96 kHz.

The resampled IR length is scaled proportionally (`ceil(256 * host_sr / 48000)`). The `activeIR` vector and the circular history buffer are both reallocated in `prepare()` — `reset()` only zeroes the history without reallocating.

## Signal Chain Position

`CabinetIR` is applied after the full `Overdrive` signal chain (post waveshaper, post tone/mid/presence EQ, post output level) and before the hard-clip ceiling of ±1.0 that the JUCE plugin applies in `processBlock`. The recommended usage is:

```text
in → [Overdrive] → [CabinetIR] → hard-clip ±1.0 → out
```

When `CabinetIR` is disabled (`setEnabled(false)`), it is a true no-op — `process()` returns immediately without touching the buffer or updating internal state.

## Implementation

Direct-form FIR convolution using a circular history buffer:

```cpp
void process(float* buffer, int numSamples) override {
    if (!enabled) return;
    const int N = (int)activeIR.size();
    for (int i = 0; i < numSamples; ++i) {
        history[(size_t)head] = buffer[i];
        float y = 0.0f;
        for (int k = 0; k < N; ++k)
            y += activeIR[k] * history[(size_t)((head - k + N) % N)];
        head = (head + 1) % N;
        buffer[i] = y;
    }
}
```

**CPU cost:** At 48 kHz with N=256 taps, direct-form convolution requires 256 multiply-adds per sample = 12.3 million multiply-adds per second. On modern hardware this is negligible. For N > 512 an overlap-add FFT convolver would be more efficient.

**History buffer:** Allocated and zeroed in `prepare()`. Zeroed (but not reallocated) in `reset()`.

## API

| Method | Description |
| --- | --- |
| `setEnabled(bool)` | Enable or disable the IR; false = true no-op bypass (default false) |
| `setType(CabinetType)` | Select the active cabinet preset; takes effect on next `prepare()` call |
| `getType()` | Returns the currently selected `CabinetType` |
| `prepare(double sr, int blockSize)` | Resamples and activates the selected IR for `sr`; reallocates history buffer |
| `reset()` | Zeroes the history buffer without reallocating |
| `process(float*, int)` | Direct-form convolution; no-op when disabled |

## JUCE Integration

Two APVTS parameters control cabinet IR:

| ID | Type | Default | Description |
| --- | --- | --- | --- |
| `od_cabinet` | Bool | false | Enable cabinet IR |
| `od_cabinet_type` | Choice (0–2) | 0 | Preset: 0 = 1×12 open-back, 1 = 4×12 closed-back, 2 = 1×12 combo |

Both are wired in `processBlock()`:

```cpp
cabinet[i].setEnabled(cabEnabled);
cabinet[i].setType(cabType);
cabinet[i].process(channelData, numSamples);
```

The editor shows a "Cabinet Type" combo box (third combo row in the Overdrive section) and a "Cabinet IR" toggle button side-by-side with "Pick Sensitivity".

## VCV Rack Integration

Two parameters control the cabinet:

| Param | ID | Description |
| --- | --- | --- |
| Panel CKSS toggle | `CABINET_PARAM` | Enable/disable cabinet IR; at x=25.4 mm, y=96 mm, labeled "CAB" |
| Right-click menu | `CABINET_TYPE_PARAM` | Cabinet preset (0–2); selected via context menu: 1×12 Open-Back / 4×12 Closed-Back / 1×12 Combo |

Applied in the per-sample process loop after `effect.process()`:

```cpp
cabinet.setEnabled(params[CABINET_PARAM].getValue() > 0.5f);
cabinet.setType(static_cast<CabinetType>((int)params[CABINET_TYPE_PARAM].getValue()));
cabinet.process(&sample, 1);
```
